Embedded Sidescroller Game with Custom Hash Table

C  ·  ESP32-C6  ·  FreeRTOS  ·  uLCD  ·  Georgia Tech

Overview

A fully playable 2D side-scrolling game built on the ESP32-C6 microcontroller, featuring Georgia Tech's mascot Buzz the Yellow Jacket. The player navigates a procedurally generated world, collecting pollen and dodging stingers, while a custom dynamic difficulty system, a top-3 leaderboard, and background music all run concurrently on resource-constrained embedded hardware.

Technical Highlights

  • Rendered a 128×128 uLCD display with real-time sprite drawing, anti-flicker caching, and frame-budgeted updates to handle 10+ simultaneous entities without tearing
  • Implemented a custom hash table with separate chaining for O(1) spatial lookup of game objects across a cyclic 60×9 tile world map with procedural panel generation
  • Integrated GPIO-driven buzzer audio with a background melody engine and sound-effect system running alongside gameplay logic via FreeRTOS tasks
  • Built a navigation-switch input system, multi-button handler, and LED strip feedback all communicating over hardware peripherals without blocking the game loop
  • Designed a gravity system, dynamic scroll speed, and three-slot persistent leaderboard stored in-memory across rounds

Skills Gained

  • Embedded C / ESP32-C6
  • FreeRTOS multitasking
  • Hash table design
  • uLCD graphics pipeline
  • GPIO & hardware peripheral integration
  • Game loop architecture

Diamond Path – Pathfinding Optimization Engine

RISC-V Assembly  ·  Georgia Tech

Overview

A pure RISC-V Assembly program that finds the minimum-cost 5-step path out of a diamond-shaped grid of 121 weighted cells using a dynamic programming approach. The solution fills cumulative costs outward from the center along four axes and four quadrant loops, then scans all 20 boundary endpoints to return the global minimum.

Technical Highlights

  • Replaced a brute-force permutation search with in-place dynamic programming: costs propagate outward along right, left, up, and down axes first, then four quadrant passes fill remaining cells by comparing two candidate predecessor costs per cell
  • Used 5-bit binary masking as a hardware-level multiplexer to evaluate path permutations without branch overhead, minimizing dynamic instruction count
  • Optimized at the ISA level against strict baseline metrics for dynamic execution length, static code size, and operand storage — all within the constraints of bare-metal RISC-V (no OS, no stdlib)
  • Implemented entirely with RISC-V integer instructions (addi, lw, sw, bge, blt, slli, jal), no pseudo-instructions, across structured labeled loops for each axis and quadrant

Skills Gained

  • RISC-V Assembly
  • Dynamic programming
  • ISA-level optimization
  • Bit manipulation
  • Low-level memory addressing

Discovery Project – Temperature-Controlled Fan System

The Problem

For my Discovery Project, I wanted to challenge myself with a hands-on system that involved real ECE concepts such as sensors, microcontroller programming, and motor control. The core question I set out to solve was: "How can sensor data be used in electronic systems to automatically control physical outputs?" Although simple, this problem pushed me to explore embedded systems and integrate multiple hardware components.

The Process

1. Empathize & Understand

I researched temperature sensors and learned that the DHT11 would be easy to interface with an Arduino. I also studied how PWM works for controlling fan speed and how motor drivers protect the microcontroller.

2. Define

The system needed to be able to:

  • Read live temperature values
  • Display the readings clearly
  • Turn a fan on when the temperature rises
  • Adjust fan speed automatically

3. Ideate

I selected the following components:

  • Arduino Uno
  • DHT11 Sensor
  • 16×2 LCD Display
  • L298N Motor Driver
  • 5V DC Fan

I planned wiring diagrams, reviewed data sheets, and learned which pins connect to the LCD, sensor, and driver.

4. Prototype

I built the first version on a breadboard:

  • Connected the DHT11 to Arduino pin 7
  • Wired the LCD using 4-bit parallel mode
  • Connected the fan to the L298N motor driver outputs
  • Programmed PWM logic to control speed based on temperature

5. Test

I tested the system by blowing warm air on the sensor and observing the LCD data and fan behavior. I also tested multiple fan types and learned how current limitations affect performance.

The Outcome

The final system successfully reads real-time temperature, displays it on the LCD, and controls the fan's speed automatically based on threshold values. The project clearly demonstrates a closed-loop control system using microcontroller logic, sensors, and output hardware.

Successes & Challenges

Successes

  • Accurate temperature readings
  • Real-time display using LCD
  • Automatic fan speed control using PWM
  • Successful integration of all hardware components

Challenges

  • Voltage drops caused the LCD to dim when the fan activated
  • A larger 5V fan required too much current for the Arduino to supply
  • The DHT11 sometimes produced sensor errors until delays were added
  • L298N voltage drop reduced fan performance

Skills Gained

  • Arduino / C++
  • DHT11 Sensor
  • LCD Interfacing
  • PWM Motor Control
  • Power Management
  • Circuit Debugging
  • System Integration
